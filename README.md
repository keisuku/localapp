# Local App Studio

**Excel / CSV / JSON をドラッグ&ドロップで取り込み、ブラウザの中だけで完結する PC 向けローカル業務アプリ基盤**

求人比較・保険案件管理・在庫管理・差分チェックなど、さまざまな業務アプリへ「モジュールを追加するだけ」で横展開できる設計です。データはすべて自分の PC のブラウザ内（IndexedDB）に保存され、**外部サーバーへの送信は一切ありません**。

|---|---|
| 技術スタック | React / TypeScript / Vite / Tailwind CSS / shadcn/ui スタイル / TanStack Table / Dexie.js (IndexedDB) / SheetJS / react-dropzone |

| 配布形態 | ① ローカル開発サーバー ② GitHub Pages ③ オフライン単一 HTML ④ PWA インストール |

## セットアップ

Node.js 20 以上（推奨 22）が必要です。

```bash
# 依存パッケージのインストール
npm install

# 開発サーバーの起動（http://localhost:5173）
npm run dev

# 本番ビルド（dist/ に出力。オフライン単一HTML版も同梱される）
npm run build

# ビルド結果の確認
npm run preview
```

## 使い方

### 初回起動

初回起動時にオンボーディング画面が表示されます。**「サンプルデータで試す」** を押すと、3 つのモジュール（汎用データ管理・求人比較・保険案件管理）にリアルなサンプルデータが投入され、すぐに全機能を体験できます。

### 基本操作

| 操作 | 方法 |
|---|---|
| ファイル取り込み | 画面のどこにでもファイルをドロップ / 上部の「インポート」ボタン |
| クリップボード取り込み | Excel でセル範囲をコピー → アプリ上で `Ctrl+V` |
| 新規追加・編集 | 「新規追加」ボタン / 行クリックで詳細ドロワー → 編集 |
| インライン編集 | テーブルのセルをダブルクリック（星評価はクリックで直接変更） |
| 検索 | 上部検索バー（スペース区切りで AND 検索） |
| フィルター | 「フィルター」ボタン → 条件を追加。名前を付けて保存も可能 |
| 一括操作 | 行のチェックボックスを選択すると下部に一括操作バーが出現 |
| ビュー切替 | ダッシュボード / テーブル / カード / カンバン |
| エクスポート | 「エクスポート」→ Excel / CSV / JSON（表示中の絞り込み結果を出力） |
| 元に戻す | 操作直後のトーストの「元に戻す」/ `Ctrl+Z`（直近 50 操作） |

### コマンドパレット（Ctrl+K）

`Ctrl+K` でコマンドパレットが開きます。モジュール移動・新規追加・インポート・エクスポート・バックアップ・テーマ切替などの主要操作に加え、**全モジュールを横断したレコード検索**ができます。`?` キーでショートカット一覧を表示します。

### スコアリング

モジュールごとに定義されたロジックでレコードを自動スコアリングします（例: 求人比較 = 年収 40 点 + リモート 20 点 + 魅力度 20 点 + 残業の少なさ 20 点）。詳細ドロワーで内訳を確認できます。

## データ保存の仕組み

- すべてのデータはブラウザの **IndexedDB**（データベース名 `local-app-studio`）に保存されます。
- 操作のたびに**即時自動保存**され、画面右下に最終保存時刻が表示されます。
- 削除・初期化などの破壊的操作は必ず確認ダイアログを経由し、実行後も「元に戻す」で復元できます。
- **注意**: ブラウザの「閲覧データの削除」でデータが消えます。設定画面から定期的に **JSON バックアップ** を保存してください。バックアップファイルをドロップするだけで復元できます。

## Excel / CSV / JSON の扱い

### 取り込み（4 ステップのウィザード）

1. **取込先の確認** — 列名からモジュールを自動判定（手動変更可）。先頭 5 行をプレビュー。
2. **列のマッピング** — ファイルの列 → モジュール項目の対応を確認。列名・同義語（例: 「年収」「給与」）から自動マッピング済み。未対応の列は「新しい項目として追加」を選ぶとノーコードで項目が生えます。
3. **データチェック** — 必須項目の空欄 / 重複（ファイル内 + 既存データ） / 異常値（範囲外の数値・解釈できない日付など）を自動抽出し、行単位でスキップするか選択。
4. **完了** — 取り込み結果の要約カードを表示。履歴は設定画面からいつでも**まとめて取り消し**可能。

### 文字コード・形式

- **CSV**: UTF-8 を優先し、失敗時は Shift_JIS として自動再解釈（Excel が出力する日本語 CSV に対応）。
- **CSV 出力**: UTF-8 BOM 付きのため Excel でそのまま文字化けせずに開けます。
- **Excel (.xlsx)**: 複数シートの場合は先頭シートを使用。日付セルは自動で `YYYY-MM-DD` に正規化。
- **JSON**: 「オブジェクトの配列」は表データとして取り込み。バックアップ形式の JSON は復元フローへ自動分岐。

## モジュールの追加方法

`src/modules/` にフォルダを 1 つ追加するだけです。**コア側のコード修正は不要**（Vite の `import.meta.glob` で自動登録されます）。

```
src/modules/wine/            ← 例: ワイン在庫管理
├── schema.ts      … フィールド定義・ステータス定義
├── sampleData.ts  … サンプルデータ
├── scoring.ts     … スコアリングロジック（任意）
├── views.ts       … テーブル列順・カード・カンバン・集計カードの設定
├── labels.ts      … 日本語ラベル・アイコン
└── index.ts       … 上記を ModuleDefinition に組み立てて default export
```

最小の `index.ts`:

```ts
import type { ModuleDefinition } from '@/core/types/module';
import { fields, statuses, defaultStatus, version } from './schema';
import { sampleData } from './sampleData';
import { views } from './views';
import { labels } from './labels';

const module: ModuleDefinition = {
  id: 'wine',           // フォルダ名と一致させる
  version,
  fields,
  statuses,
  defaultStatus,
  views,
  labels,
  sampleData,
};
export default module;
```

ポイント:

- **`fields` の `aliases`** に同義語を書いておくと、インポート時の列名自動マッチングが賢くなります（例: `aliases: ['銘柄', 'ワイン名']`）。
- **`unique: true`** を付けた項目は取り込み時の重複検出キーになります。
- **`scoring`** を定義すると、テーブル・カード・ダッシュボードに自動でスコア列が現れます。
- アイコンは `src/components/common/Icon.tsx` に登録済みの lucide アイコン名から選びます（必要なら 1 行追加）。
- **コードを書く前の試作**は、設定画面の「項目のカスタマイズ」でノーコードに項目を追加 → 「雛形をコピー」で `schema.ts` のコードとして書き出すのが近道です。
- 既存モジュールのフィールドを変更する場合は `version` を上げ、`migrate(data, fromVersion)` で旧データを変換できます。

`generic`（汎用）・`jobs`（求人比較）・`insurance`（保険案件管理）の実装がそのまま参考例になります。

## GitHub Pages での公開方法

1. リポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に変更（初回のみ）。
2. `main` ブランチへ push すると `.github/workflows/deploy.yml` が自動でビルド・デプロイします。
3. 公開 URL: `https://<ユーザー名>.github.io/localapp/`

リポジトリ名を変える場合は `vite.config.ts` の `base`（`'/localapp/'`）を合わせて変更してください。ルーティングはハッシュ方式（`#/jobs/table`）のため、Pages 特有の 404 対策は不要です。

## オフライン版の配布

`npm run build` を実行すると、JS/CSS をすべてインライン化した **自己完結の単一 HTML**（`dist/local-app-studio-offline.html`）が生成されます。

- 公開版アプリの **設定画面 →「オフライン版をダウンロード」** からも取得できます。
- ファイルをダブルクリックするだけで起動し、**ネット接続ゼロ**で全機能（保存・取込・出力）が動作します。
- 社内 PC への配布や、ネットワーク分離環境での利用に便利です。
- データはブラウザ + ファイルの場所（オリジン）ごとに保存されるため、ファイルを移動するとデータ領域も変わります。移動前に JSON バックアップを取ってください。

また PWA に対応しており、公開版をブラウザの「アプリをインストール」からデスクトップアプリのように利用することもできます。

## ライセンス / 開発メモ

- すべての書き込みは `src/core/db/mutations.ts` を経由し、逆操作を捕捉して Undo を実現しています。
- UI コンポーネントは shadcn/ui 互換のスタイルで `src/components/ui/` に同梱（外部レジストリ非依存）。
- `npm run typecheck` で型検査、`npm run build` でビルドできれば壊れていません。
